# SGT-Gen 技术报告（01）：问题定义与总体方案

## 1. 项目定位
SGT-Gen（Source Graph Tree Generation）是一个面向 SFT 数据构建的工程化框架，目标是从**源文档**出发，同时利用**知识图结构**与**层级树结构**生成可控、高覆盖、可审查的训练样本。

## 2. 核心问题
- 仅靠自由生成，样本可控性差，易同质化。
- 仅靠图结构，层级意图与任务分布不均衡。
- 仅靠树结构，事实约束弱，跨主题关系链缺失。
- 缺少统一的质量闸门，难以在批量生成时稳定产出。

## 3. SGT-Gen 核心思路
- `Source`：以源文档作为事实锚点，避免闭门造车。
- `Graph`：实体-关系图负责事实覆盖与关系推理路径。
- `Tree`：意图树负责主题规划、层级分布与难度梯度。
- `Generation`：在“树定目标、图供上下文、源可回溯”的约束下生成 QA/评测样本。

## 4. 流程总览
1. 文档读取与切分。
2. 实体关系抽取并构建图谱。
3. 意图树加载或生成。
4. 分区与子图采样（含层级分区）。
5. 模式化生成（atomic/聚合/multi_hop/cot/hierarchical）。
6. 规则与模型双重质量校验。
7. 结构化落盘与任务追踪。

## 5. 可交付输出
- SFT 训练样本（JSON/JSONL，多格式）。
- 评测样本集（evaluation dataset）。
- 覆盖率与分布指标报告（taxonomy coverage、dimension distribution）。
- 任务级运行日志与 token 统计。
