## 0. 摘要

SGT-Gen（Source Graph Tree Generation）是一个面向 SFT 数据生成的工程化框架，核心目标是将“源文本真实性”“图结构关系性”“树状层级性”三种信号统一到一条可复现流水线。系统采用前后端分离：前端（Vue）负责任务配置与过程管理，后端（FastAPI）负责任务编排与结果管理，核心算法位于 `graphgen/`。

相较于历史版本，本轮重构重点包括：
- 统一核心命名为 SGT-Gen，并在代码层保持向后兼容别名；
- 清理遗留 Gradio 应用层，仅保留后端/CLI 必需的任务与工作目录模块；
- 强化批处理并发逻辑，降低“批次残留导致挂起”的风险；
- 重写实验计划并细化可执行命令、结果结构与判据；
- 明确项目短板并给出可借鉴项目清单与落地路径。

## 1. 系统目标与设计原则

### 1.1 系统目标

1. 从多格式源文档抽取高可信知识结构；
2. 按可控策略生成多类型 QA 数据；
3. 通过规则与评估反馈提升数据质量；
4. 形成可复现实验闭环（数据、命令、结果、结论）。

### 1.2 设计原则

- **真实性优先**：答案可回溯源文档/子图；
- **结构性优先**：显式利用图与树关系；
- **可扩展优先**：模块化替换生成器、分区器、评估器；
- **工程稳定优先**：并发控制、缓存与容错明确。

---

## 2. 架构与关键模块

### 2.1 三层架构

1. **前端层**（`frontend/`）：任务创建、配置管理、审阅与状态可视化；
2. **后端层**（`backend/`）：任务状态机、执行编排、产物落盘；
3. **核心算法层**（`graphgen/`）：文档处理、图构建、图分区、QA 生成、评估。

### 2.2 核心流水线

1. **读取与切分**：多格式 Reader + tokenizer 对齐；
2. **知识构建**：抽取实体关系并写入图存储；
3. **（可选）理解评估**：测量知识点难度或模型不确定性；
4. **图分区与采样**：按 BFS/DFS/ECE/层级等方法构造子图批次；
5. **QA 生成与格式化**：多模式输出（atomic/聚合/multi-hop/cot/hierarchical）。

### 2.3 本轮关键工程改造

1. **命名统一**：核心类改名为 `SGTGen`，保留 `GraphGen = SGTGen`；
2. **旧前端清理**：移除 Gradio 入口与依赖，保留任务管理/工作目录工具；
3. **并发修复**：批请求管理器改为“可排空批次”策略，避免残留请求长期等待。

## 3. 实验设计（概要）

详细实验命令与结构见：`experiments/EXPERIMENT_PLAN.md`。

### 3.1 三阶段评估

1. **可行性实验（Phase-A）**：小规模快速验证“能跑通、可提升”；
2. **系统实验（Phase-B）**：多域完整对比与统计检验；
3. **消融实验（Phase-C）**：验证 Tree / Hierarchical / Critic / Sampling 贡献。

### 3.2 评估维度

- 数据质量：可答率、事实一致率、格式合规率、重复率；
- 覆盖能力：实体覆盖、关系覆盖、层级覆盖；
- 训练价值：小规模 SFT 后目标任务提升；
- 工程效率：总耗时、吞吐、单位样本成本。

---

## 4. 当前短板与改进方向（概要）

### 4.1 当前短板

1. 分区与采样策略在不同数据分布下仍需更稳健的自适应；
2. QA 质量判别主要依赖规则，统计判据仍可更细；
3. 端到端观测链路不够细粒度，排障效率受限；
4. 跨域迁移配置成本仍偏高。

### 4.2 优先改进方向

1. 引入覆盖驱动采样回路（coverage-aware sampler）；
2. 增强评估报告模板（自动输出分层结论与异常点）；
3. 建立标准化“生成失败案例库”并驱动 prompt/规则迭代；
4. 增强实验自动化脚本，降低人工整理成本。
